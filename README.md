### 设计模式原则：
1、单一职责

2、接口隔离

3、依赖倒转：面向接口编程，抽象不应该依赖细节，细节应该依赖抽象

4、里氏替换：尽量不要覆盖父类方法

5、开闭原则OCP

6、迪米特法则：最少知道原则；直接朋友

7、合成复用原则：尽量使用合成、聚合的方式，而不是使用继承
